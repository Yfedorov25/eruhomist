Stack locked: Next.js 15 + Lenis + GSAP, no WebGL. #1 goal = phone calls. Perf budget sacred. Decisions: (1) repo placement · (2) hero day→night engine · (3) frame-sequence tour decode/memory · (4) interactive floorplan · (5) location map with unknown distances.

Council Verdict

Where the council agrees

  • Hero → Option A, layered next/image + GSAP opacity (5/5). Day = full-opacity LCP element; fade night 0→1, ease:none. The "muddy middle" is a myth — opaque photos don't grey out. verified: quadro hero already ships this
  • Map → never invent distances. A fake/vague map reads as "hiding something" and poisons trust in the price; a missing map goes unnoticed. Escalated to legal liability on a $270k sale. spec already mandates this
  • Tour is THE product-defining risk — de-risk first; on a decode miss render the nearest decoded frame, never black.
  • Repo → apps/nahirna (reversible in minutes). already at root; wire into monorepo like apps/quadro
  • The phone number is the product — tel: at top, bottom, and a following pill.

Where the council clashes

  • Floorplan technique. Hand-redrawn SVG (Expansionist) vs PNG+hotspots (Executor) vs 3-4 hotspots only (1st Principles). The buyer settles it: indifferent to draw technique, cares about correct numbers + the rooms that sell. → PNG/SVG underlay + positioned hotspot buttons. areas already exact for all 7 rooms → label all 7, lead the eye to terrace 29.83m² + master 16.65m²; no hand-redraw, no gallery scope
  • The scope war. Adopt the Expansionist's conversion psychology (end-of-tour sticky CTA at the golden-hour climax, magnetic call pill, one honest scarcity line) — reject its engineering ambition (oversized decode window, a speculative "reusable villa engine," an all-room gallery) before villa #1 ships.

Blind spots the peer review caught

  • No reduced-motion / no-JS / iOS Low-Power fallback for the tour (untested — scrub may not even fire under Low Power).
  • No total-decode-failure degraded path → must fall back to a static golden-hour hero + CTA.
  • No measurable perf gate on a named mid-tier Android (merge-blocking).
  • No call-conversion instrumentation — the worst omission for an Amplitude-driven team. Log which section was on screen when the call fired.
  • Frame colour consistency at the two crossfade seams (three separate AI clips spliced).
  • build-frames.sh is the contract the decoder depends on — and it was unrun. caught a real defect: FPS=30 → ~542 frames, 3× over target; corrected to FPS=10 → 181 frames
  • Distances are a content blocker the client owns — but one true line ("own riverbank · 0 min") already exists; get ONE headline distance and express POIs as time.

The recommendation, decision by decision

1 · Repo: Move into apps/nahirna; inherit the Vercel + Telegram-lead pipeline from apps/quadro; keep the public/ path exact (case-sensitive FS).
2 · Hero: Two next/image layers, GSAP opacity scrub, day = priority LCP, fade night 0→1, ease:none. No canvas.
3 · Tour: canvas drawImage; createImageBitmap; sliding window current±8 (~16-24 decoded); .close() on eviction; decode-ahead; cap ~200MB; nearest-decoded on miss. Fallbacks: reduced-motion → static first frame; decode-fail/Low-Power → static golden-hour hero + CTA.
4 · Floorplan: Flatten the PDF to an underlay; positioned hotspot buttons over real zones; label from exact areas; warm-accent active, dim rest; aria-label + keyboard; mobile tap + docked panel.
5 · Map: Static stylized frame + villa pin now (fastest LCP, no Leaflet). POIs as time, verified only ("own riverbank · 0 min" ships today); unverified POIs behind a flag, not a visible TODO.
Cross-cutting: call instrumentation from day 1 (tel: everywhere + Amplitude per-section event); magnetic call/callback pill after the water story; perf gate on a real mid-tier Android; every section ships reduced-motion + mobile fallback.

The one thing to do first

Run scripts/build-frames.sh, then validate its output (count 150-180, correct widths, no zero-byte) and eyeball the two crossfade seams for colour/exposure jumps — before writing any overlay code. The frames didn't exist yet (source is 3 mp4s), and the script's default FPS overshot the target 3×. Done during this session: master built, FPS corrected to 10 → 181 frames.

Full advisor responses

The Contrarian flaw-hunter · co-strongest

(1) In-place forks Vercel config; moved assets must keep the public/ path or next/image 404s on Vercel's case-sensitive FS. (2) Hero A, but the "muddy middle" is the wrong fear — opaque photos don't grey out; the real flaw is full-res layer overdraw + lerp-0.08 overshoot; use ease:none, fade only the night layer. (3) "Where the project dies": mobile tab-kill ~400MB; createImageBitmap guarantees decoded RAM; window 12-15 + .close() or leak GPU; mobile 60 frames; clamp velocity; render last-decoded on miss. (4) A, but watch exact-area drift; trace + label from PDF numbers. (5) C, hard — invented distances on a legal sale = liability; hide behind a flag; Leaflet = LCP grenade.

The First Principles Thinker scope discipline · co-strongest

(1) False framing, 5-min reversible — apps/nahirna. (2) You invented the muddy middle — day full-opacity beneath, night 0→1 on top; a buyer can't perceive a blend curve. (3) The memory math is fantasy — never decode 180 at once; window current±8, ~20-30MB; decode-starvation is the only real risk. (4) Gold-plating — nobody clicks 14 zones on a phone; 3-4 hotspots max on the rooms that sell. (5) Static frame now (fastest LCP). Net: 2/3/4 over-scoped — the phone call is the product.

The Expansionist upside · biggest blind spot (5/5)

(1) Move → a reusable cinematic-villa engine, villa #3 becomes a fork. (2) Layered, never canvas; a third mid-dusk golden-hour frame = the most screenshot-able moment. (3) Window ~40, decode-ahead; end on the riverbank at golden hour + a sticky "Book a private viewing" the instant it completes = peak emotional arousal. (4) Hand-redrawn; click a room → area + photo, doubles as a gallery. (5) Static map with POIs as time. Plus: a persistent magnetic call/callback pill; one scarcity line. Reviewers: upside-drunk — adds expensive engineering before the centerpiece is de-risked.

The Outsider — the buyer settled the floorplan + map clashes

Doesn't know the jargon, neither will the buyer. (1) Internal plumbing — stop discussing. (2) Only notice if broken/muddy ("cheap"). (3) The scroll-film is the ONE thing that could make me call — but a black-flash/stutter screams "amateurs, trust them with $270k?"; broken centerpiece worse than none. (4) I care only that I tap a room and see 18m² + what it overlooks; numbers correct. (5) Leave the map OUT rather than fake/vague — made-up distances = "hiding something" → I distrust the price. Phone number top + bottom + a following button.

The Executor order-of-operations

De-risk Decision 3 first — build nothing else until it's proven. Validate build-frames.sh (count 150-180, 1920w, no zero-byte, eyeball seams), then the overlay. Sliding window ~24 decoded + .close(); nearest-ready on miss; cap ~200MB. (1) Move to apps/nahirna. (2) Hero A, no debate. (4) PNG underlay + positioned hotspot buttons — an afternoon; hand-redraw = 2 days zero gain. (5) Static frame + pin + TODO now. Order: validate frames → move → scaffold → hero → 01-04 → overlay → map stub → facts/CTA/footer.
