# 🅰 СИСТЕМА ЧИТАБЕЛЬНОСТІ (база для ВСІХ секцій — зробити першою)

## Проблема
Світлий/тонкий текст тоне на світлих ділянках медіа (небо, скло, тюль). Вбиває «дорого».

## Принцип
Три рівні захисту. Застосовувати НАЙСЛАБШИЙ, який спрацював для конкретного місця.
Hero H1 / ціна / CTA — гарантований контраст ЗАВЖДИ.

## Реалізація — 3 переюзабельні утиліти

### Рівень 1 · `Scrim` (градієнт, НЕ суцільна плашка)
Компонент-оверлей поверх медіа, під текстом. Лінійний градієнт, прозорий на чистому медіа.
```tsx
// components/ui/Scrim.tsx — варіанти напрямку
// top:    linear-gradient(to bottom, rgba(15,15,14,.45) 0%, transparent 120px)
// bottom: linear-gradient(to top,    rgba(15,15,14,.78) 0%, transparent 45%)
// left:   linear-gradient(to right,  rgba(15,15,14,.7)  0%, transparent 55%)
```
- `pointer-events:none`, `position:absolute inset:0`, `z-index` між медіа і текстом.
- Параметри: `direction` (top|bottom|left|right), `strength` (0–1 → множник альфи).

### Рівень 2 · `TextShelf` (диптих: текст на шоколаді поруч із фото)
Layout-патерн: текст живе на суцільній панелі `#1a1510` (або `#100c08`), фото — поруч, не під ним.
Використати на 02 (див. ТЗ 02). Нечитабельність стає неможливою.

### Рівень 3 · `safe-zone` (правило вибору кадру/позиції)
Не компонент — правило: текст кладемо на ТЕМНУ частину кадру. Якщо кадр світлий усюди —
переходимо на Рівень 2. У alt/доборі кадрів обирати ті, де є темна зона під текст.

## Застосування по екранах (точкове)
- **Hero надзаголовок:** `Scrim direction=top strength=0.5` + текст тепло-білий `#EDEAE3`
  (НЕ золото). Золото лишається тільки в H1.
- **Hero H1:** `Scrim direction=bottom strength=0.6` під блоком H1 (рятує світлі простінки,
  невидимий на темному фасаді).
- **02:** повністю на `TextShelf` (Рівень 2) — див. ТЗ 02.
- **06:** фон уже темний — додатковий scrim не потрібен.
- **CTA-пігулка «Замовити дзвінок»:** не чіпати, читається завжди.

## Definition of Done
- [ ] `Scrim` і `TextShelf` — переюзабельні компоненти в `components/ui/`.
- [ ] Надзаголовок Hero читабельний на день і ніч (перевір Playwright-скрін обох станів).
- [ ] H1 читабельний на світлих простінках між колон.
- [ ] Жоден scrim не виглядає як «плашка» (градієнт, не суцільний прямокутник).
- [ ] Контраст тексту проти фону ≥ WCAG AA для основного тексту.

# 🅱 ТЗ 02 · АРХІТЕКТУРА — «Зроблено для того, хто залишається» (ПОВНА ПЕРЕРОБКА)

## Що прибрати
Старий прийом «window-reveal крізь скло» (фото інтер'єру, що виїжджає з-за фасаду) — ВИДАЛИТИ
повністю. Він читався як зламана верстка. Жодного виїжджання збоку.

## Мета й емоція
Продати ЯКІСТЬ матеріалів і ДОВГОВІЧНІСТЬ («зроблено надовго»). Перехід день→вечір показує:
вдень дім строгий каменем, надвечір — теплий світлом. Емоція: «це збудовано для того, хто
залишається» (архетип Творець/Спадщина).

## Патерн
Ядро: **text-shelf диптих** — текст-полиця ліворуч (шоколад) + кадр-вікно праворуч.
У кадрі-вікні — **day→night dissolve на scroll-scrub** (НЕ морф, а розчинення: день fade-out,
ніч fade-in, + легкий scale 1.0→1.04 для «дихання»). Підтримка: тихий рядок деталей знизу.

## Layout (desktop)
```
┌─────────────────────────────────────────────┐
│ [ПОЛИЦЯ #1a1510 ~38%]   [КАДР-ВІКНО ~62%]     │
│  надрядок: матеріали     ░░ day ░░            │
│  H2 (день/ніч стан)      ░░ ↓ dissolve ↓ ░░   │
│  абзац про матеріали     ░░ night ░░          │
│  місток (курсив)                              │
├─────────────────────────────────────────────┤
│ [Клінкер]  [Колона]  [Дах]  ← тихі тези       │
└─────────────────────────────────────────────┘
```

## Ассети (наявні — звір у /public/images/)
- `exterior-terrace-columns.webp` — крупний денний кадр (тераса/колони). **Кадр-вікно, стан «день».**
- `exterior-night-1.webp` (або `-2`) — нічний фасад з підсвіткою. **Кадр-вікно, стан «ніч».**
- ⚠️ Денний і нічний кадри МОЖУТЬ бути різного ракурсу. Тому НЕ робимо піксельний crossfade-морф —
  робимо **dissolve** (opacity 0↔1 + легкий scale). Невеликий зсув ракурсу читається як «той
  самий дім в інший час», не як глюк. Це навмисне рішення, не баг.
- Якщо нічний кадр зовсім інший за композицією — обрати найближчий за ракурсом до денного.

## Поведінка скролу
- Секція pin на ~150vh (luxury-темп, не поспішати).
- 0→100% progress кадру-вікна: `day` opacity 1→0, `night` opacity 0→1 (dissolve, scrub).
  Паралельно обидва шари scale 1.0→1.04.
- Полиця: H2 у стані «день» (текст про камінь) на старті; на ~50% progress м'яко змінюється
  на стан «ніч» (текст про світло) — fade-through, не різко.
- Рядок деталей знизу: stagger fade-in коли секція в центрі viewport (по черзі, 1.5s power3.out).
- Текст полиці завжди читабельний (шоколадний фон) — scrim не потрібен.

## Текст (з COPY_REBUILD §02 — взяти ДОСЛІВНО)
- Надрядок: `матеріали`
- H2 день: `Удень цей дім — про камінь.`
- H2 ніч: `А коли темніє — про світло.`
- Абзац: `Темна клінкерна цегла й білий камінь на колонах. Такі матеріали не оновлюють кожні
  пʼять років — вони просто гарнішають. Цей дім будували для того, хто залишається.`
- Місток (курсив): `Один поверх. Між Вами і ранковою кавою на терасі — жодної сходинки.`
- Деталі (3 тези, повні, НЕ обрізані): Клінкер / Колона / Дах — точні формулювання з COPY_REBUILD.

## Технічні обмеження
- Dissolve через opacity двох absolute-шарів + scale (тільки transform/opacity).
- reduced-motion → показати один кадр (нічний, як драматичніший) + повний текст полиці, без dissolve.
- mobile: полиця зверху, кадр під нею (стек); dissolve лишається (легкий), pin коротший або без pin.
- Кадри: next/image, lazy (below-the-fold), оптимізований WebP.

## Definition of Done
- [ ] Старий «крізь скло» прийом повністю прибраний.
- [ ] Dissolve день→ніч плавний на scrub, без «наїжджання» фото.
- [ ] Текст полиці читабельний завжди (Playwright-скрін).
- [ ] Деталі знизу — повні тези, не обрізані.
- [ ] reduced-motion + mobile стек працюють.
- [ ] GSAP cleanup.

# 🅳 ТЗ ЛОАДЕР · «Рівень води» (НОВИЙ)

## Що прибрати
Плоский текстовий лоадер. Замінити на бренд-момент.

## Мета й емоція
Перша емоція й обіцянка якості. Концепт «власний берег» з першої секунди. Спокій, «дорого».
Маскує preload перших ~20 кадрів hero (технічна функція + перформанс).

## Патерн
Темна вода піднімається 0→100% знизу вгору (CSS clip-path / height + легка SVG-хвиля з брижами).
На 100% — «розлив» угору (curtain-reveal), що відкриває hero.

## Поведінка
- Фон `#0F0F0E`. Вода — темно-бірюзовий `#16323d` з тонкою лінією поверхні `#3a6575`.
- Рівень 0→100% за ~1.8–2.5s, easing power2.inOut (luxury, не поспішати).
- Поверхня води — легка SVG-хвиля (1–2 синуси, дуже повільна анімація, transform тільки).
- Слово над поверхнею (тихо, тепло-біле): `Власний берег`.
- Лічильник скраю (тонко, золото): `0 → 100`.
- На 100%: вода «розливається» вгору 0.8s power4.out → переходить у hero (з'являється hero-day).
- Лоадер НЕ має сам гальмувати: якщо ассети готові раніше — мінімум 1.2s показу (щоб встигли
  відчути момент), потім reveal.

## Текст (з COPY_REBUILD)
- `Власний берег` (над водою). Жодного «Завантаження».

## Технічні обмеження
- Тільки transform/opacity/clip-path. Без важких бібліотек.
- reduced-motion → пропустити анімацію, одразу статичний hero-day (без води).
- Не блокувати LCP: hero-day-кадр має вантажитись паралельно; лоадер маскує саме preload кадрів.
- mobile: те саме, легша хвиля.

## Definition of Done
- [ ] Вода піднімається 0→100% плавно, брижі живі.
- [ ] На 100% розлив угору відкриває hero без стрибка (CLS=0).
- [ ] «Власний берег» читабельний над водою.
- [ ] reduced-motion → миттєвий hero без води.
- [ ] Лоадер не додає затримки понад необхідну для preload (~1.2–2.5s стеля).
- [ ] GSAP cleanup; жодного залипання лоадера якщо ассет не вантажиться (таймаут-fallback).
