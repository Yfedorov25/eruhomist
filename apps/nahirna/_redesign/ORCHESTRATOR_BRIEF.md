# ТЗ для Клода-оркестратора — асети для «Дім на Нагірній» (senior++ апгрейд)

> ТИ — Клод-оркестратор генерації ассетів. Твоя робота: дослідити НАЙКРАЩІ поточні AI-інструменти
> (nano-banana / Gemini для зображень, Kling AI / Veo для відео, інші — на твій вибір), і за цим ТЗ
> видати ТОЧНИЙ shopping-list генерацій (промпти, seed, роздільність, fps, тривалість, pin-кадри) +
> consistency-правила. Людина запустить генерації й покладе ассети в репо. Потім Claude Code (інший
> чат) реалізує. ТИ НЕ пишеш код — ти даєш план генерації + точні промпти + правила консистентності.
>
> Це рішення прогнано через раду 5 радників + chairman (звіт `_redesign/council-assets-20260604.md`).

## F1 — ПЛАВНИЙ 3D-ТУР + ДЕНЬ/НІЧ
═══════════════════════════════════════════════════════════════════════════════
**Поточне:** 181 WebP-кадр з 18.6с (3 Kling-кліпи) @ ~10fps → ВИМІРЯНО РВЕТЬСЯ (той самий кадр 6
кроків скролу). Декодер ЗДОРОВИЙ (sliding-window, nearest-decoded, нема чорного) — проблема в ДЖЕРЕЛІ.

**ВЕРДИКТ: НЕ суцільний 540-кадровий прохід (дрифт), НЕ ~900 кадрів/стан (LCP). А: 4 дискретні
locked-шоти, РІЗАТИ між ними (не морфити), нативно ≥24fps, ~120 кадрів сумарно.** Плавність =
нативний 24fps + крос-фейд движка + чистий різ між шотами. Хребет: скрол = «день, що ти проживаєш»,
фінал = НІЧ-з-води.

**SHOPPING-LIST (точні промпти під кожен; pin на реальні рендери):**
| Шот | Рух | Start pin → End pin | Трив. | fps | Res | Стан |
|---|---|---|---|---|---|---|
| S-A | ворота → фасад | `exterior-day-1` → `exterior-terrace-columns` | 5s | 24 | 1920w | день |
| S-B | фасад → тераса | `exterior-terrace-columns` → `terrace` | 5s | 24 | 1920w | день/golden |
| S-C | **тераса → вода (головний)** | `water-terrace-golden` → `water-finale` | 5s | 24 | 1920w | golden→останнє світло |
| S-D | **НІЧ-З-ВОДИ (hero/OG)** | нова нічна версія `water-finale` → hold | 4s | 24 | 1920w | **найвища важність: теплі вікна, чорна річка** |

**ДЕНЬ/НІЧ:** кожен шот у ДВОХ станах з ІДЕНТИЧНИХ Kling-камер-keyframes (ті самі pin, день/ніч).
**Нічні pin через nano-banana:** подай денний рендер, *«той самий будинок, ті самі меблі, та сама
камера, ніч, тепле внутрішнє світло, стримане золото»*, **seed-lock + reference-image на шот.**
Claude Code крос-фейдить день↔ніч; скрол-кінець = НІЧ.

**S-D «ніч-з-води» — НАЙВАЖЛИВІШИЙ ассет** (рада одноголосно): незавершений будинок не продає меблі,
але «світло ввімкнене, ти вдома, річка — чорне скло внизу» продає ЖИТТЯ. Hero-якість + 2-3с мікро-
луп. Reuse як OG/share.

**PERF (жорсткий):** ≤120 desktop / ≤96 mobile сумарно. Eager-20 ≤4.5MB. Повний desktop ≤~22MB
progressive. Якщо ніч роздуває → ніч крос-фейд-парами, не другою повною секвенцією.

**DISCLOSURE:** S-D і інтер'єрні beat несуть P0-візуалізація-рядок.

## CROSS-CUTTING (обов'язкове)
═══════════════════════════════════════════════════════════════════════════════
1. **Consistency Bible** — один документ (villa-identity sheet + 14 рендерів reference + матеріали +
   camera-locked day/night пари + disclosure-правило + rejection-test). Людина ЗАТВЕРДЖУЄ перед
   будь-яким батчем. Додається до КОЖНОГО промпту.
2. **Provenance** — кожен ассет: {pin-рендер, seed, промпт, дата}. Усі = swap-in плейсхолдери
   (реальний будинок добудують → перегенерувати). Закладай ДЕШЕВУ регенерацію.
3. **Legal disclosure** — required-ассет, ПЕРЕД будь-яким новим інтер'єром.
4. **Бюджет-cap** — 3 спроби/шот. Не тримається 3× → fallback на статичний pin-рендер.
5. **Acceptance** — LCP<2.5/CLS<0.1/60fps на mid-tier, ТІЛЬКИ prod-білд.
