# ФАЗА 5 / КРОК 1 — Промпт для Claude Code: 3D-карта району

> Точки обрані, асети готові. Це повне ТЗ на побудову <DistrictMap/>.
> Передати Claude Code разом із data/poi.js (вже створено).

---

```
Прочитай recipes/R10_3d-district-map.md, teardowns/ERA_real-estate.md (розділ /3d-map),
data/poi.js і CLAUDE.md.

Контекст: будуємо killer-feature нерухомості — інтерактивну 3D-карту району, рівня
era.estate /3d-map. Це окрема секція сайту QUADRO. Точки вже обрані (8 шт у data/poi.js).

АСЕТИ (вже в проєкті):
- public/assets/district_map.webp — стилізована top-down підкладка району (темна преміум-тема,
  озеро ліворуч-внизу, оливковий парк у центрі, латунна сітка вулиць, спокійний центр).
- POI-фото підтягуються пізніше; поки використовуй заглушку-градієнт, якщо фото нема (photo:false).

⚠️ КРИТИЧНО (урок з ARCHITECTURE_AUDIT): WebGL тут УЖЕ падав чорним екраном і був видалений.
Карта = ІЗОЛЬОВАНИЙ R3F-canvas, який МОНТУЄТЬСЯ ТІЛЬКИ в межах своєї секції (lazy, on intersection)
і має context-guard: при webglcontextlost або помилці ініціалізації → автоматичний fallback
на статичну 2D-карту (div з district_map.webp + HTML-маркери поверх), БЕЗ чорного екрану.
Падіння карти НЕ валить решту сайту.

ПОБУДОВА <DistrictMap/> (R3F + @react-three/drei):

1. СЦЕНА:
   - Площина (PlaneGeometry) з текстурою district_map.webp, лежить горизонтально (rotation.x=-PI/2).
   - Легкий нахил камери (PerspectiveCamera), початковий вид — м'яка 3/4 перспектива згори.
   - Підсвічений бокс QUADRO у центрі (layout {x:0,z:0}): невисокий паралелепіпед,
     emissive тепла теракота (~#e0703c), м'яке пульсування emissiveIntensity (1.5–3s цикл).
   - Краї площини йдуть під fade у фон (vignette/градієнт), щоб не було різкого обрізу.

2. POI-МАРКЕРИ (з data/poi.js, позиція = layout.x/z, масштабовані на розмір площини):
   - кожен маркер: тонке вертикальне emissive-кільце/стовпчик + drei <Html> лейбл
     (назва + «N хв» з walkMin/driveMin). Колір лейбла з CATEGORIES[cat].
   - пульсація кільця синхронна, але субтильна. Hover → лейбл яскравішає, кільце росте трохи.
   - НЕ підписувати координати; тільки назва + час у дорозі.

3. ФІЛЬТР КАТЕГОРІЙ (HTML-оверлей поверх canvas, не в WebGL):
   - кнопки-чіпи з CATEGORIES (💧 Озеро, 🌳 Парки, 🍽 Ресторани, 🏋 Спорт, 🛍 ТРЦ, 🧸 Садочки).
   - клік по чіпу → toggle видимості маркерів цієї категорії (fade out/in, GSAP 0.4s).
   - активні чіпи підсвічені; дефолт — усі активні.

4. ВЗАЄМОДІЯ:
   - OrbitControls: enablePan=false, enableDamping=true, dampingFactor=0.08,
     polarAngle обмежений (не дати «впасти під карту»), minDistance/maxDistance ~18/45.
   - клік на маркер → GSAP flyTo: камера плавно (1.2s, power3.out) наближається до точки,
     одночасно відкривається HTML-модалка: фото (або градієнт), назва, відстань (км),
     час у дорозі, рейтинг (зірки), наратив text із poi.js. Фон сторінки dim.
   - закриття модалки → камера повертається у вихідний вид (1.2s power3.out).

5. PRELOADER карти (поки вантажиться текстура + сцена):
   - зворотний роллер цифр 100→0 (translateY-стопка цифр), elegant, у стилі ERA.
   - зникає fade коли сцена готова.

6. КОМПАС (опційно, якщо легко):
   - маленька стрілка-індикатор, що обертається = -azimuthalAngle камери.

FALLBACK / A11Y:
   - mobile АБО prefers-reduced-motion АБО no-WebGL → статична 2D-карта:
     div з district_map.webp (object-fit cover) + абсолютно спозиційовані HTML-маркери
     (по layout.x/z → %), клік → та сама модалка. Без OrbitControls, без flyTo.
   - context-guard (з Фази 1): втрата WebGL → миттєвий перехід у цей 2D-режим.
   - усі маркери досяжні з клавіатури (focusable, Enter відкриває модалку).

ТЕМП / СТИЛЬ:
   - усі анімації easing power3.out/expo.out, тривалість 1.2–1.5s (luxury, не швидко).
   - палітра з CLAUDE.md: темний фон, тепла теракота акцент, латунь.
   - текст модалок з poi.js (українською, вже написаний — НЕ переписувати).

PERFORMANCE:
   - lazy-mount canvas тільки коли секція в viewport (IntersectionObserver).
   - текстура district_map стиснута (≤1.5 МБ). next/image для POI-фото.
   - unmount canvas коли секція пішла з viewport (звільнити WebGL-контекст) — економія пам'яті.
   - ціль 60fps desktop; на mobile одразу 2D-режим.

РЕЗУЛЬТАТ:
   Поверни <DistrictMap/> + Playwright-тест: (а) карта рендериться, (б) клік на маркер
   відкриває модалку з правильним текстом, (в) фільтр категорій ховає/показує маркери,
   (г) симуляція webglcontextlost → перехід на 2D без чорного екрану.
   Зупинись на оцінку — покажу результат, оцінимо проти ERA /3d-map.
```
